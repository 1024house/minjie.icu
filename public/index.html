中华田园敏捷宣言

我们一直在摸索多快好省的软件开发方法，身体不行的同时也福报他人。由此我们建立了如下价值观：

  • 皮实和本分 高于 技术和创新
  • 领导的需求 高于 用户的价值
  • 加快开发 高于 减少 bug
  • 做完发版 高于 按时下班

也就是说，尽管右边有价值，但我们更重视左边。

敏捷下乡

如果你对团队管理、项目管理、代码质量、单元测试、远程办公有困惑，比如：

  • 如何考核程序员的工作量？
  • 程序员估工时总是不准确，怎么办？
  • 如何增效，不增加人手，而提高 30% 产出？
  • 如何降本，除了裁员，能否有别的办法降低 30% 成本？
  • 业务部门提需求都说自己是「高优先级」，怎么办？
  • 随着项目功能越来越多，bug 也越来越多，怎么办？能不能罚款？
  • 软件开发为什么总是延期？能否打造像工厂流水线那样时间精确的软件工厂？
  • 老项目用的技术是十年前的，废弃有漏洞了，怎么办？比如 .NET Core，要废弃重写吗？
  • 项目越来越大，回归测试越来越久，导致发版速度降低，怎么办？
  • 听说单元测试挺好的，应该让测试人员去学，还是开发去学？据说学习成本很高？
  • Java 8 将于 2023 年底废弃，要提前几年升级吗？
  • 公司接外包，订单回款之前，能不能不发工资？
